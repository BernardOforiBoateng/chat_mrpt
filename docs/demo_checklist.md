# ChatMRPT Demo Preparation Checklist

## Pre-Demo Setup (Day Before)

### Technical Preparation
- [ ] Test ChatMRPT on AWS (http://3.137.158.17)
- [ ] Ensure TPR workflow is functioning
- [ ] Test both data pathways (CSV+Shapefile AND TPR-only)
- [ ] Clear browser cache and test fresh login
- [ ] Prepare backup local instance if needed
- [ ] Test internet connection at venue

### Data Preparation
- [ ] Have sample CSV ready (Kano_plus.csv or similar)
- [ ] Have shapefile ZIP ready
- [ ] Have TPR Excel file ready (NMEP data)
- [ ] Create backup copies on USB drive
- [ ] Test data uploads work smoothly

### Presentation Materials
- [ ] Review demo script 3-5 times
- [ ] Time each section (aim for 15-20 minutes total)
- [ ] Prepare visual aids/slides if needed
- [ ] Print backup script notes
- [ ] Prepare business cards/contact info

## Demo Day Setup (1 Hour Before)

### Technical Check
- [ ] Boot up system 30 minutes early
- [ ] Test ChatMRPT is accessible
- [ ] Clear any previous session data
- [ ] Open all necessary tabs/windows
- [ ] Turn off notifications/updates

### Environment Setup
- [ ] Check projector/screen connection
- [ ] Test audio if using microphone
- [ ] Arrange seating for good visibility
- [ ] Have water ready
- [ ] Phone on silent

### Mental Preparation
- [ ] Review opening hook
- [ ] Practice key transitions
- [ ] Remind yourself: You're solving their problem
- [ ] Deep breathing exercises
- [ ] Positive visualization

## During Demo Reminders

### Opening (First 2 Minutes)
- [ ] Make eye contact with audience
- [ ] Deliver hook with confidence
- [ ] Watch for engagement signals
- [ ] Adjust energy to room

### Main Demo (10-15 Minutes)
- [ ] Keep pace steady, not rushed
- [ ] Pause after key points
- [ ] Check audience understanding
- [ ] Show enthusiasm for features
- [ ] Handle any technical issues calmly

### Closing (2-3 Minutes)
- [ ] Summarize key benefits
- [ ] Make clear call to action
- [ ] Leave contact information
- [ ] Thank audience
- [ ] Open for questions

## Post-Demo Actions

- [ ] Collect feedback/questions
- [ ] Note any technical issues
- [ ] Follow up with interested parties
- [ ] Document what worked well
- [ ] Plan improvements for next demo

## Emergency Protocols

### If ChatMRPT is Down:
1. Stay calm and professional
2. Acknowledge the issue briefly
3. Switch to backup slides/screenshots
4. Focus on value proposition
5. Offer to schedule live demo later

### If Data Won't Upload:
1. Have pre-loaded session ready
2. Explain this is unusual
3. Continue with existing data
4. Emphasize normal reliability

### If Internet Fails:
1. Use mobile hotspot backup
2. Show local instance if available
3. Use screenshots/recordings
4. Focus on business benefits

## Key Talking Points to Remember

1. **Cost Savings**: $50,000 consultant vs ChatMRPT
2. **Time Savings**: 3 months vs minutes
3. **Accessibility**: WhatsApp-simple interface
4. **Two Pathways**: Full data OR just TPR
5. **Empowerment**: Your team becomes the experts

## Success Metrics

- Audience asks "How can we get this?"
- Questions about implementation, not feasibility
- Requests for follow-up meetings
- Excitement about specific features
- Understanding of ROI
